\chapter{Einleitung}

Softwaretests sind ein Grundpfeiler für die Qualitätssicherung von Anwendungen. Für die
Tests werden Testdaten spezifiziert, anhand derer das Verhalten der Software geprüft
wird. Vor allem bei Datenbank-basierten Anwendungen können solche Testdaten sehr
umfangreich und komplex werden. In diesem Umfeld spricht man auch von der
Modellierung von Testdaten.

Der Grund für diese Komplexität liegt darin, dass nicht nur Daten beschrieben,
sondern auch Beziehungen von Datensätzen ausgedrückt werden müssen. Besonders bei Systemen mit 
komplexen Tabellen-Schemata kann die Modellierung unübersichtlich werden.

Für den Tester ist eine unübersichtliche Modellierung aus verschiedenen Gründen ein Problem.
Einerseits macht sie die Modellierung fehleranfälliger. Andererseits ist es schwerer,
die modellierten Daten zu erfassen und zu verstehen. Oftmals werden deshalb für verschiedene
Tests dieselben Daten verwendet, damit der Tester nur einen Satz von Testdaten verstehen
muss.

\section{Zielsetzung}

Die Modellierung von Testdaten soll vereinfacht werden. Dazu soll eine Modellierungssprache
entwickelt werden, die sowohl leicht verständlich als auch leicht zu erlernen ist. Besonderer
Fokus liegt auf der Modellierung von Beziehungen. Die Lösung soll einfach zu benutzen sein
und sich in Entwicklungsumgebungen wie Eclipse integrieren lassen.

Darüber hinaus sollen sich Testdaten automatisch generieren lassen. Auch hier soll besondere
Aufmerksamkeit auf die Generierung von Beziehungen zwischen Datensätzen gelegt werden.
